2. VIP Rooms — Architecture & Concepts
2.1  What Is a VIP Room?
A VIP Room is a private, real-time group chat space created by a creator. Think of it as a live backstage pass — a place where paying fans can chat directly with the creator and each other in real time. Rooms can be tied to a specific event, a video drop, a Q&A session, or simply kept open as an ongoing community space. Only users who have paid the entry fee (or been granted free access by the creator) can join the chat.
Each creator can have multiple rooms — e.g. a general community room, an event room for a music drop, a study group room.
Rooms have a configurable capacity (default 500 members, max 2,000 in Phase 1).
Messages are persisted for 30 days. The last 200 messages are sent to late-joining members as chat history.
Creators can pin messages, mute members, and kick members at any time.
Viewers can tip the creator directly inside the room — tips trigger a celebration animation visible to all members.
Campus Creator Codes (Section 7) allow university/campus-specific creators to generate invite codes giving free or discounted entry to verified students.
2.2  Room Access & Entry Fee
ℹ️
Entry fees are one-time per room — not recurring. Once a user has paid to join a room, they are a permanent member and can rejoin at any time without paying again (unless the creator resets membership, which is an admin-level action). Free rooms (entry_fee_ghs: 0.00) require no payment but still require the user to explicitly join.
entry_fee_ghs: 0.00 — Free room. Any authenticated user can join instantly with no payment.
entry_fee_ghs: > 0.00 — Paid room. User must complete MoMo payment before receiving an access token.
Creator earnings from entry fees: 85% (same as tips). Platform fee: 15%.
The creator always has access to their own room regardless of entry fee.
Subscribers (Section 4 of Tipping & Wallet doc) get free entry to all of a creator's paid rooms while their subscription is active.

2.4  Real-Time Connection Flow
Step-by-step flow from app open to active chat:
User navigates to a VIP Room in the app. If the room requires an entry fee and the user has not paid, they see the entry fee screen. They initiate payment via POST /api/v1/rooms/{room_id}/access (Section 4.1) and complete the MoMo flow.
Once payment is confirmed (or for free rooms, immediately), the app calls GET /api/v1/rooms/{room_id}/access/token (Section 4.3) to get a short-lived room_access_token.
The Expo app creates a Socket.IO connection: const socket = io('wss://api.tamkko.app', { auth: { token: accessToken, room_access_token: roomAccessToken } })
The server verifies both tokens. The JWT identifies the user. The room_access_token confirms they have paid entry. If either is invalid, the server emits an error event and disconnects.
The app emits room:join with the room_id. The server adds the socket to the room channel, emits room:joined back with the current room state and last 200 messages, and broadcasts a presence:update to all other members.
The user is now in the live chat. Outgoing messages use socket.emit('message:send', payload). Incoming messages, reactions, tips, and presence updates arrive as socket.on() listeners.
When the user leaves (navigates away or backgrounds the app), the app calls socket.emit('room:leave') then socket.disconnect(). The server broadcasts member:left and presence:update to remaining members.

4. Room Access & Entry (REST)
ℹ️
Entry to a paid VIP Room requires two steps: (1) Pay the entry fee via MoMo (REST — Section 4.1–4.2). (2) Exchange the payment confirmation for a short-lived room_access_token (REST — Section 4.3), then use that token to authenticate the Socket.IO connection. Free rooms skip step 1 and go straight to step 2.
4.1  Pay Entry Fee & Request Access Token
👤
User Story:  As a viewer who wants to join a paid VIP Room, I want to pay the entry fee using my MTN MoMo account so I become a permanent member of the room.
POST
/api/v1/rooms/{room_id}/access
Pay Room Entry
Auth Required
Yes — Authorization: Bearer {access_token}
Free rooms
For free rooms (entry_fee_ghs: 0.00), send an empty request body {}. No MoMo fields needed. Returns 200 with room_access_token immediately.
Subscribers
Active subscribers to the creator get free entry. Send empty body {} — backend checks subscription status automatically.
Already joined
Returns 409 ALREADY_ROOM_MEMBER with existing access token — no new charge.
📤  Request Body (paid rooms only)
{
"momo_number": "+233241234567",
"momo_provider": "MTN"
}

Created — Entry fee payment initiated
✅  Response — Paid room (poll required)

"room_id": "room_4a5b6c7d",
"amount_ghs": "5.00",
"status": "pending",
"ussd_prompt_sent": true,

"entry_id": "entry_free_3i4j5k",
"room_id": "room_4a5b6c7d",
"status": "granted",

5. Real-Time Chat (Socket.IO)
ℹ️
All Socket.IO events use the /rooms namespace. Connect to: wss://api.tamkko.app/rooms. Every event payload must include room_id. The server validates the room_access_token on connect and rejects invalid or expired tokens immediately.
5.1  Connect & Authenticate
Establish the Socket.IO connection with both the JWT access_token (identifies the user) and the room_access_token (proves room entry was paid). Both are passed in the auth object at connection time.
Expo App — Establish Connection
{
"// Install: npm install socket.io-client": "",
"const socket = io('wss://api.tamkko.app/rooms', {": "",
"  auth: {": "",
"    token: accessToken,": "",
"    room_access_token: roomAccessToken": "",
"  },": "",
"  transports: ['websocket'],": "",
"  reconnection: true,": "",
"  reconnectionAttempts: 5,": "",
"  reconnectionDelay: 2000": "",
"})": ""
}
On successful connection, immediately emit room:join (Section 5.2). If authentication fails, the server emits an error event with code SOCKET_AUTH_FAILED and disconnects.

👤

Client → Server  (emit: message:send)

Max length

Muted users
Muted members receive error code MEMBER_IS_MUTED — their message is not sent.

In-room tips work identically to video tips (see Tipping & Wallet module) except they are sent inside a VIP Room context. When a tip is confirmed, the tip:received Socket.IO event fires to all room members — triggering a celebration overlay animation visible to everyone in the room.
