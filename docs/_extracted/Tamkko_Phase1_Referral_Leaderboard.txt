2. Referral System — Architecture & Concepts
2.1  How the Referral System Works
The referral system is a creator-to-creator growth mechanism. Every Tamkko user automatically receives a unique referral code and a shareable link on account creation. When a new user registers using that link or code, a permanent referral relationship is recorded. The referrer then earns a small percentage of Tamkko's commission on all tips the referred creator receives — for as long as they both remain on the platform.
Every user gets a unique referral code at registration — no action needed from the user.
Referral link format: tamkko.app/join/{referral_code} — e.g. tamkko.app/join/KWAME8X
A user can only have one referrer. Once a referral relationship is set at registration, it cannot be changed.
Referral relationships are one level deep in Phase 1 — there is no multi-level (MLM) structure. Creator A refers Creator B. Creator A earns rewards on B's tips. Creator B does NOT earn rewards on creators they refer (that is a Phase 2 feature).
The referral reward comes out of Tamkko's commission share — not from the referred creator's earnings. Creator B always receives 85% of tips regardless of whether they were referred.
Referral rewards are credited to the referrer's wallet as soon as the referred creator's tip payment is confirmed by the Hubtel webhook.
2.2  Reward Calculation — The Commission Split
Detailed breakdown of a GHS 10.00 tip to a referred creator:
Recipient
Amount (GHS 10.00 tip)
Description
Referred Creator (Creator B)
GHS 8.50 (85%)
Creator's earnings — unchanged by referral system
Tamkko Platform Commission
GHS 1.50 (15%)
Total platform cut before referral split
— Referrer Reward (Creator A)
GHS 0.75 (5% of commission)
5% of the GHS 1.50 commission = GHS 0.075, rounded to GHS 0.08 minimum
— Tamkko Net Revenue
GHS 1.42 (remaining)
Platform keeps the rest after paying referrer
⚠️
The reward rate (5% of commission) is configurable by admins and may be adjusted over time. See Section 7.4. The frontend should always display the rate fetched from the API — never hardcode it.
2.3  Campus Ambassador System
Campus Ambassadors are a special tier of referrers — creators who are verified students or campus community leaders that actively promote Tamkko on their campus. Ambassadors get an elevated referral reward rate and a special badge on their profile. In Phase 1, ambassador status is applied manually by admins after reviewing an application. The Campus Creator Codes feature in the VIP Rooms module is a separate but complementary feature — ambassadors typically generate campus codes for their VIP rooms as well.
Ambassador reward rate: 8% of Tamkko's commission (vs 5% for regular referrers) — configurable by admin.
Ambassador badge displayed on profile next to the creator's username.
Ambassador leaderboard: separate Campus Leaders leaderboard showing top referrers per university.
Ambassadors can see how many students joined Tamkko through their referral link.
2.4  Referral Reward Flow (Step by Step)
Creator A shares their referral link: tamkko.app/join/KWAME8X on WhatsApp and social media.
Creator B taps the link. The app opens (or prompts to install). The referral_code KWAME8X is stored locally in expo-secure-store and also passed as a query parameter to the registration screen.
Creator B completes registration. The referral_code is sent in the registration request body. The backend validates the code, looks up Creator A's account, and creates a referral relationship record linking B to A.
Creator B uploads videos and starts receiving tips from their followers.
A fan sends Creator B a GHS 10.00 tip. The Hubtel webhook confirms the payment.
The tip webhook handler dispatches a Celery task: calculate_referral_reward(tip_id).
The Celery task looks up Creator B's referral relationship, finds Creator A as the referrer, calculates the reward (5% of GHS 1.50 commission = GHS 0.075, rounded up to GHS 0.08), and credits Creator A's wallet with GHS 0.08.
A referral_reward transaction is recorded in Creator A's wallet transaction history with subtype: referral_reward and a reference to the original tip.
Creator A receives a push notification: 'You earned a referral reward! @creator_b received a tip and you earned GHS 0.08.'
Creator A can withdraw referral rewards along with all other wallet earnings — no separate withdrawal needed.

3.3  Apply Referral Code at Registration
ℹ️
The referral_code field is added to the existing registration endpoint (POST /api/v1/auth/register) documented in the Authentication module. This section documents only the referral-specific behaviour. The full registration request body is in the Auth document.
Referral field added to POST /api/v1/auth/register:
📤  Registration Request Body (referral field only)
{
"phone_number": "+233241234567",
"password": "SecurePassword123!",
"username": "new_creator_gh",
"referral_code": "KWAME8X"
}
referral_code is optional. If omitted, the user registers without a referrer. If provided, the backend validates it and records the relationship. An invalid code returns a validation warning but does NOT block registration — the user is registered without a referral relationship.
📤  Referral section of registration response
{
"referral": {
"code_applied": "KWAME8X",
"referred_by": {
"username": "kwame_creates",
"display_name": "Kwame Asante"
},
"relationship_created": true
}
}

5. Leaderboard
ℹ️
Leaderboards are cached in Redis as sorted sets and recalculated every 15 minutes. They are public — no auth required to view. Three leaderboard categories in Phase 1: Top Referrers (most creators referred), Fastest Growing Networks (most referral earnings in the past 30 days), and Campus Leaders (top referrers per university/campus, sourced from ambassador applications). Each leaderboard shows the top 100 positions.
